## 2. 브레이크포인트 전략

```
──────────────────────────────────────────────────────────
0px      323px  373px          540px  620px       841px+
│         │      │              │      │            │
│ (없음)  │Z6커버│Z3~5커버       │현재앱 │Z6펼침       │Z3~5펼침
│         │narrow│standard-fold │max   │fold-expand │fold-wide
──────────────────────────────────────────────────────────
```

### 미디어쿼리 3단계

```css
/* ① 좁은 커버 (Z Fold 6 커버) */
@media (max-width: 360px) { /* narrow-fold */ }

/* ② 펼침 진입 (Z Fold 6 펼침 이상) */
@media (min-width: 600px) { /* fold-expand */ }

/* ③ 넓은 펼침 (Z Fold 3~5 펼침) */
@media (min-width: 820px) { /* fold-wide */ }
```

> **실무 적용**: ①은 긴급 버그픽스, ③은 ②의 확장 처리. 핵심은 **②(600px+)** 분할 레이아웃.

## 3. 접힌 상태 (커버 스크린) UX 이슈 & 해결책

### 3-1. 현재 문제 목록

| 컴포넌트 | 문제 | 우선순위 |
|---------|------|---------|
| `EmpVehicle` 주간 출근지 | `minWidth: 580px` 고정 → 가로 스크롤 불가피 | 🔴 HIGH |
| `EmpSalary` 5컬럼 테이블 | 323px에서 컬럼 극단적으로 압축 | 🟡 MID |
| 통계 2×2 카드 (`gridTemplateColumns: "1fr 1fr"`) | 323px에서 각 셀 ~150px → 숫자 폰트(24~32px) 잘림 | 🟡 MID |
| 일보 상세 2컬럼 출퇴근 인풋 | `flex: 0 1 140px` → 323px에서 오버플로 | 🟢 LOW |
| FAB(+ 버튼) `right: 20px` | 커버에서도 동일 — 문제 없음 | ✅ OK |

### 3-2. 커버 스크린 해결 원칙

**원칙 A: 가로 스크롤 허용 (표 형태)**
- 주간 출근지 그리드: 가로 스크롤 컨테이너(`overflow-x: auto`) 명시적 안내 추가
- "← 스크롤" 힌트 텍스트 또는 그라디언트 페이드 처리

**원칙 B: 단일 컬럼 리플로우 (카드/폼)**
- 통계 2×2 카드: `@media (max-width: 360px)` → `gridTemplateColumns: "1fr"` (2행 4개)
- 급여 5컬럼 테이블: 3컬럼으로 축소 (날짜 / 급여 / 상태), 나머지 → 상세 탭

**원칙 C: 폰트 스케일 다운**
- 32~36px 숫자 → 24px (커버)

## 4. 펼친 상태 (내부 스크린) — 분할 레이아웃 전략

### 4-1. 글로벌 레이아웃 변경

```
┌──────────────────────────────────────────────────────┐
│  BottomNav → Left Sidebar (펼침 시 자동 전환)          │
├────────────┬─────────────────────────────────────────┤
│            │  Header (sticky, 타이틀만)                │
│  SIDEBAR   │─────────────────────────────────────────│
│  (64~200px)│                                         │
│            │        CONTENT AREA                     │
│  [아이콘]  │        (max-width: 해제 또는 확장)        │
│  + 라벨    │                                         │
│            │                                         │
│  [로그아웃]│                                         │
└────────────┴─────────────────────────────────────────┘
```

- `≥ 600px`: BottomNav 숨김, Left Sidebar(64px 아이콘+라벨 세로) 표시
- `≥ 820px`: Sidebar 200px로 확장 (텍스트 라벨 표시)
- `maxWidth`: 600px+ → 제한 해제, content는 `flex: 1`로 남은 공간 활용
- Header: 로그아웃 버튼 → Sidebar 하단으로 이동

## 7. 인터랙션 전환 패턴

### 탭 전환 (BottomNav → Sidebar)
- `≥ 600px`: `position: fixed; left: 0; top: 0; bottom: 0; width: 64px`
- 탭 라벨: `≥ 820px`에서 텍스트 표시, 그 이하는 아이콘만
- 현재 탭 indicator: 하단 3px 빨간 바 → **좌측 3px 빨간 바**로 전환

### 목록→상세 전환
- `< 600px`: 풀스크린 전환 (현재 방식 유지)
- `≥ 600px`: 우측 패널 인라인 표시 (뒤로가기 불필요)
- 선택 항목 없을 때 우측 패널: "항목을 선택하세요" 빈 상태

### 모달
- `< 600px`: 현재 방식 (overlay 모달, 풀폭 카드)
- `≥ 600px`: 인라인 패널로 대체 (모달 제거)
